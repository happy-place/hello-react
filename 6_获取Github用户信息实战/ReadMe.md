## 获取Github用户信息实战
```aidl
1.创建模拟访问github用户信息接口，能正确请求到数据就发送真实数据，否则发生模拟数据；
2.guithub由于已经通过CROS配置，允许跨域访问，故能直接访问，而配置CROS不安全，因此使用配置代理策略，访问模拟服务接口；
3.划分页面区域为搜索区Search、列表区List；
4.移植静态页，先全部迁入App.js 和 App.css，对对于没有使用样式的外层div可以省略，替换className，style，每改一步，
就检查一次，全局搜索查找样式被哪些标签使用，拆分样式；
5.数据由Search获取，交给List展示，由于不使用PusSub进行组件通信，因此要进行状态提升，将数据存入App的state;
6.Search页点击按钮，才发起查询，触发事件对象和事件操作对象不同，因此属于不可控事件，需要使用ref标记输入框；
7.List页首次进入需要有提示语句，发起查询需要有Loading状态，查询结束，请求成功这展示数据，请求失败则展示异常信息，此处用到连续三元表达式进行分支操作
```
